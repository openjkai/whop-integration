# Contributing to Whop Integration

Thank you for considering contributing to this project! Here's how you can help:

## Development Setup

1. Fork the repository
2. Clone your fork: `git clone https://github.com/yourusername/whop-integration.git`
3. Install dependencies: `npm install`
4. Create a branch: `git checkout -b feature/your-feature-name`
5. Make your changes
6. Test thoroughly
7. Commit: `git commit -m "Add: your feature description"`
8. Push: `git push origin feature/your-feature-name`
9. Open a Pull Request

## Code Style

- Use TypeScript for all new files
- Follow the existing code structure
- Use meaningful variable and function names
- Add comments for complex logic
- Keep functions small and focused

## Testing

Before submitting a PR:

1. Test locally with `npm run dev`
2. Verify all features work:
   - Checkout flow
   - Subscription status
   - Cancellation
   - Reactivation
3. Check for TypeScript errors: `npm run build`
4. Test with actual Whop API (use test keys)

## Pull Request Guidelines

- Provide a clear description of changes
- Link any related issues
- Include screenshots for UI changes
- Update documentation if needed
- Ensure all tests pass

## Areas for Contribution

- Additional payment providers
- UI/UX improvements
- Better error handling
- More comprehensive tests
- Documentation improvements
- Example integrations (Auth0, NextAuth, etc.)
- Mobile responsiveness
- Accessibility improvements

## Questions?

Open an issue for discussion before starting major changes.
